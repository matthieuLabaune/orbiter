# Glossaire — Orbiter

| Terme | Définition |
|-------|-----------|
| **Requirement (REQ)** | Exigence fonctionnelle ou technique, tracée vers des tests et du code |
| **V&V** | Vérification & Validation. Vérification = conforme aux specs. Validation = répond au besoin réel |
| **Module** | Découpage fonctionnel du projet (équivalent composant système) |
| **ADR** | Architecture Decision Record — journal des décisions techniques |
| **Risk Score** | Score FMEA = Impact × Probabilité × (6 - Détectabilité) |
| **Baseline** | Snapshot immuable de l'état du projet à un instant T |
| **Deploy Readiness** | Revue GO/NO-GO avant déploiement, inspirée NASA Flight Readiness |
| **Context Brief** | Briefing contextuel généré pour les agents IA |
| **Anomalie** | Comportement inattendu, pas forcément une violation d'exigence |
| **Non-conformité (NC)** | Violation d'une exigence existante → fix obligatoire |
| **Défaut (DEF)** | Le système ne fonctionne plus → hotfix critique |
| **Lesson Learned** | Capitalisation d'un apprentissage lié à un module/requirement |
| **Traçabilité** | Capacité à remonter de n'importe quel artefact vers le besoin utilisateur |
